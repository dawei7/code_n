## Problem statement

Given an array of unique integers `nums`, return all possible subsequences (subsets).
A subsequence of an array is a new array that is formed from the original array by deleting some (can be none) of the elements without disturbing the relative positions of the remaining elements.
The solution set must not contain duplicate subsets. Return the solution in any order.

**Input:** An integer array `nums`.
**Output:** A 2D array of integers representing all subsequences.

## When to use it

- To generate every single possible combination of elements.
- The fundamental "Pick / Not Pick" recursion tree applied to array accumulation instead of mathematical sums.

## Approach

**1. The "Take it or Leave it" Decision Tree:**
We process the elements of the input array one by one from left to right.
For any given element `nums[i]`, we have two mutually exclusive choices:
- **Take it:** We add `nums[i]` to our current building subsequence.
- **Leave it:** We do NOT add `nums[i]` to our subsequence.

For an array of `[1, 2, 3]`:
- Do we take 1? Yes -> Do we take 2? Yes -> Do we take 3? Yes. (Subsequence: `[1, 2, 3]`)
- Do we take 1? Yes -> Do we take 2? No -> Do we take 3? Yes. (Subsequence: `[1, 3]`)
- Do we take 1? No -> Do we take 2? No -> Do we take 3? No. (Subsequence: `[]`)

Every single possible combination is naturally generated by exhaustively exploring both branches of this binary decision tree!

**2. The Recursion State:**
We need a recursive function `recurse(index, current_path)`.
- `index` tracks which element in `nums` we are currently making the decision on.
- `current_path` is an array (or list) containing the elements we have "Taken" so far.

**3. The Base Case:**
When `index` reaches `len(nums)`, we have made a Take/Leave decision for EVERY SINGLE element in the array.
We have reached the bottom of the tree! The `current_path` is now a fully formed, valid subsequence. We append a *copy* of `current_path` to our global results array, and `return` back up the tree.

## Algorithm

<summary>Show Algorithm</summary>

```python
"""Optimal solution for recursion_03: Print Subsequences.

For each position, branch on include/exclude. The recursion
tree has 2^n leaves; each leaf is a distinct subsequence.
Sorted so the verify can do a plain equality check.
"""


def solve(s, n):
    result = []

    def helper(i, path):
        if i == n:
            result.append("".join(path))
            return
        # Exclude s[i].
        helper(i + 1, path)
        # Include s[i].
        path.append(s[i])
        helper(i + 1, path)
        path.pop()

    helper(0, [])
    result.sort()
    return result
```

## Walk-through

`nums = [1, 2]`. `results = []`.
Start: `recurse(0, [])`.

1. **`recurse(0, [])`** (Decision for `1`):
   - TAKE branch: Append `1`. Path=`[1]`. Call `recurse(1, [1])`.
2. **`recurse(1, [1])`** (Decision for `2`):
   - TAKE branch: Append `2`. Path=`[1, 2]`. Call `recurse(2, [1, 2])`.
3. **`recurse(2, [1, 2])`**:
   - `index == 2` (Base Case). Append `[1, 2]` to `results`. Return.
4. Back in `recurse(1)`: Backtrack! `pop()`. Path is back to `[1]`.
   - LEAVE branch: Do nothing. Path=`[1]`. Call `recurse(2, [1])`.
5. **`recurse(2, [1])`**:
   - `index == 2` (Base Case). Append `[1]` to `results`. Return.
6. Back in `recurse(0)`: Backtrack! `pop()`. Path is back to `[]`.
   - LEAVE branch: Do nothing. Path=`[]`. Call `recurse(1, [])`.
7. **`recurse(1, [])`** (Decision for `2`):
   - TAKE branch: Append `2`. Path=`[2]`. Call `recurse(2, [2])`.
8. **`recurse(2, [2])`**:
   - Base Case. Append `[2]`. Return.
9. Back in `recurse(1)`: Backtrack! `pop()`. Path is back to `[]`.
   - LEAVE branch: Do nothing. Path=`[]`. Call `recurse(2, [])`.
10. **`recurse(2, [])`**:
    - Base Case. Append `[]`. Return.

Tree fully explored!
`results = [[1, 2], [1], [2], []]`. ✓

## Complexity

| | Time | Space |
|---|---|---|
| **Best** | $O(N \cdot 2^N)$ | $O(N)$ |
| **Average** | $O(N \cdot 2^N)$ | $O(N)$ |
| **Worst** | $O(N \cdot 2^N)$ | $O(N)$ |

For every element in an array of size N, we branch exactly twice (Take or Leave). This creates a binary recursion tree with exactly 2^N leaf nodes. At every leaf node (the base case), we copy the `current_path` array, which takes $O(N)$ time.
Therefore, total time complexity is $O(N \cdot 2^N)$. This algorithm fundamentally explodes for N > 20.
Space complexity is $O(N)$ to hold the `current_path` array and the recursion call stack depth. (Excluding the $O(N \cdot 2^N)$ space required to hold the output `results`).

## Variants & optimizations

- **Subsets II (Handling Duplicates):** If the input array has duplicates like `[1, 2, 2]`, the recursion tree will generate identical subsets. You fix this by sorting the input array first, and skipping the "TAKE" branch for any element that is identical to the previous element IF the previous element was "LEFT" behind!
- **Bitwise Subsequences (`bit_manipulation_04`):** You can generate all 2^N subsets completely iteratively using bitwise operations! Loop i from 0 to 2^N-1. The binary representation of i (e.g. `101`) acts as a mask: include `nums[0]`, leave `nums[1]`, include `nums[2]`.
