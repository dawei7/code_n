## Problem statement

Given a string `s` of length N, calculate the total number of mathematically distinct (unique) substrings within the string.
For example, for `s = "ababa"`, the substring `"aba"` appears twice, but should only be counted once.

**Input:** A string `s`.
**Output:** An integer representing the count of distinct substrings.

## When to use it

- To measure the exact entropy or complexity of a string.
- This is the canonical textbook example of why the LCP Array was invented.

## Approach

**1. The Mathematical Maximum:**
How many total substrings exist in a string of length N?
A string of length 5 has:
- 5 substrings of length 1
- 4 substrings of length 2
- 3 substrings of length 3
- 2 substrings of length 4
- 1 substring of length 5
The total number of substrings is the sum of the first N integers: N(N+1)/2.
For N = 5, the maximum possible number of substrings is 5 x 6 / 2 = 15.

**2. Subtracting the Duplicates:**
If all characters in the string are unique (e.g. `"abcde"`), the answer is exactly N(N+1)/2.
But if there are repeated characters, some substrings will be generated multiple times.
We need to subtract the exact number of duplicates from our maximum total.
How do we mathematically find every single duplicated substring without hashing them all?

**3. The LCP Array Insight:**
The Suffix Array perfectly sorts all suffixes alphabetically. This forces all identical prefixes to be grouped together sequentially!
The LCP (Longest Common Prefix) Array tells us exactly how many characters are shared between a suffix and the suffix immediately preceding it in the sorted array.
If `lcp[i] = 3` (e.g. `"anana"` and `"ana"` share `"ana"`), it mathematically means that the 3 substrings `"a"`, `"an"`, and `"ana"` were ALREADY generated by the previous suffix! They are duplicates!
Therefore, the total number of duplicate substrings generated across the entire string is EXACTLY the sum of all elements in the LCP Array!

**4. The Final Formula:**
`Distinct Substrings = Total Substrings - Sum(LCP Array)`
\text{Distinct} = \frac{N(N+1)}{2} - \sum_{i=0}^{N-1} \text{LCP}[i]

```python
"""Optimal solution for suffix_04: Count Distinct Substrings.

Count the number of distinct non-empty substrings
"""


def solve(s, n):
    """Count distinct non-empty substrings via suffix array."""
    if n == 0:
        return 0
    # Build the suffix array naively.
    sa = sorted(range(n), key=lambda i: s[i:])
    # Build the LCP array (Kasai-style, but the naive
    # version suffices for small n).
    rank = [0] * n
    for i, idx in enumerate(sa):
        rank[idx] = i
    lcp = [0] * n
    h = 0
    for i in range(n):
        if rank[i] > 0:
            j = sa[rank[i] - 1]
            while i + h < n and j + h < n and s[i + h] == s[j + h]:
                h += 1
            lcp[rank[i]] = h
            if h > 0:
                h -= 1
    # Sum of (n - sa[i] - lcp[i]) for i in 0..n-1.
    total = 0
    for i in range(n):
        total += n - sa[i] - lcp[i]
    return total
```

## Walk-through

`s = "banana"`, N = 6.
Theoretical maximum substrings: 6 x 7 / 2 = 21.

1. Build Suffix Array: `[5, 3, 1, 0, 4, 2]`
   - `0: "a"`
   - `1: "ana"`
   - `2: "anana"`
   - `3: "banana"`
   - `4: "na"`
   - `5: "nana"`
2. Build LCP Array comparing each suffix to the one above it:
   - `LCP[0] = 0` (No neighbor above `"a"`)
   - `LCP[1] = 1` (`"ana"` shares `"a"` with `"a"`)
   - `LCP[2] = 3` (`"anana"` shares `"ana"` with `"ana"`)
   - `LCP[3] = 0` (`"banana"` shares nothing with `"anana"`)
   - `LCP[4] = 0` (`"na"` shares nothing with `"banana"`)
   - `LCP[5] = 2` (`"nana"` shares `"na"` with `"na"`)
   LCP Array: `[0, 1, 3, 0, 0, 2]`.
3. Sum of LCP Array: 0 + 1 + 3 + 0 + 0 + 2 = 6.
4. Total distinct substrings: 21 - 6 = 15.

Return `15`. ✓

## Complexity

| | Time | Space |
|---|---|---|
| **Best** | $O(N \log N)$ | $O(N)$ |
| **Average** | $O(N \log N)$ | $O(N)$ |
| **Worst** | $O(N \log N)$ | $O(N)$ |

Building the Suffix Array takes $O(N log^2 N)$ or $O(N \log N)$ depending on the chosen sorting implementation.
Building the LCP array using Kasai's Algorithm takes $O(N)$.
Summing the array takes $O(N)$.
The time complexity is bottlenecked entirely by the Suffix Array generation: $O(N \log N)$.
Space complexity is $O(N)$ for the Suffix and LCP arrays.

## Variants & optimizations

- **Suffix Tree ($O(N)$):** A Suffix Tree naturally condenses duplicate prefixes into shared edges. To count distinct substrings using a Suffix Tree, you literally just sum up the lengths of all the edges in the tree! This takes $O(N)$ time if built using Ukkonen's Algorithm, but the code is infinitely harder to write than this formula.
- **Trie ($O(N^2)$):** If the string is short (< 1000 chars), you can just insert all $O(N^2)$ substrings into a standard Trie. A Trie natively collapses identical prefixes into the same nodes! The number of distinct substrings is exactly equal to the total number of Nodes in the Trie (minus the root).

## Real-world applications

- **Information Theory:** In data compression limits, analyzing the exact ratio of Distinct Substrings vs Total Substrings mathematically proves the absolute maximum compression ratio theoretically achievable for a specific file (e.g. determining if a file is highly compressible or essentially random noise).
