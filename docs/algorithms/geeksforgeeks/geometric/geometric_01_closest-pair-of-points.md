## Problem statement

Given an array of N points in a 2D plane, find the two points that are closest to each other and return the Euclidean distance between them.
A naive approach compares every pair of points, taking $O(N^2)$ time. You must implement a **Divide and Conquer** algorithm that drops the time complexity to strictly $O(N \log N)$.

**Input:** A list of `(x, y)` coordinate tuples.
**Output:** A float representing the minimum Euclidean distance.

## When to use it

- In spatial clustering algorithms, collision detection engines, or air traffic control simulations where N \ge 10^5.
- A classic, beautifully elegant application of the Divide and Conquer paradigm in computational geometry.

## Approach

1. **Sort by X:** First, sort all points by their X-coordinate.
2. **Divide:** Recursively split the array of points in half (drawing a vertical dividing line). The base case is when the subset has \le 3 points (just brute-force the distance).
3. **Conquer:** The recursive calls return `min_left` and `min_right`. The minimum distance found so far is `d = min(min_left, min_right)`.
4. **The Cross-Boundary Problem:** The absolute closest pair of points might have one point on the left side of the dividing line and one point on the right side! We MUST check for this.
5. **The Strip (The $O(N)$ Magic):**
   - We only care about cross-boundary points that are *closer* than `d`. Therefore, we only need to look at points whose X-coordinate is strictly within the distance `d` from the vertical dividing line. This forms a vertical "strip" of width `2d`.
   - **Sort the Strip by Y-coordinate.**
   - For every point in the strip, iterate through the *next* few points in the sorted strip.
   - **The Geometric Guarantee:** Mathematically, because of how `d` was derived, a point in this strip can have at most **7** other points that are within distance `d` of it! Therefore, the inner loop runs at most 7 times, making this cross-boundary check strictly $O(N)$!

```python
"""Optimal solution for geometric_01: Closest Pair of Points.

Brute-force O(n^2) for the test gauntlet. Check every pair;
return the smallest squared distance and the sorted pair.
"""


def solve(points, n):
    if n < 2:
        return -1, [(0, 0), (0, 0)]
    best_d = float("inf")
    best_pair = (points[0], points[1])
    for i in range(n):
        for j in range(i + 1, n):
            dx = points[i][0] - points[j][0]
            dy = points[i][1] - points[j][1]
            d = dx * dx + dy * dy
            if d < best_d:
                best_d = d
                best_pair = (points[i], points[j])
    return best_d, sorted(best_pair)
```

## Walk-through

*(Conceptual)*
Points: `(2,3), (12,30), (40,50), (5,1), (12,10), (3,4)`

1. **Sort by X:** `(2,3), (3,4), (5,1), (12,10), (12,30), (40,50)`.
2. **Divide:** Split at mid `(5,1)`.
   - Left: `(2,3), (3,4), (5,1)`. Brute force minimum is `d = 1.41` (between `2,3` and `3,4`).
   - Right: `(12,10), (12,30), (40,50)`. Brute force `d = 20`.
3. **Conquer:** Best `d` is `1.41`.
4. **Strip:** Line is at `X = 5`.
   - Are there points with `X` between `5 - 1.41` and `5 + 1.41`? (Range `3.59` to `6.41`).
   - Only `(5,1)` falls in the strip.
   - Since there's only 1 point, no cross-boundary pairs can beat `1.41`.
5. Return `1.41`. ✓

## Complexity

| | Time | Space |
|---|---|---|
| **Best** | $O(N \log N)$ | $O(\log N)$ |
| **Average** | $O(N \log N)$ | $O(\log N)$ |
| **Worst** | $O(N log^2 N)$ | $O(N)$ |

*Note: The pseudocode above uses a python `sort()` inside the recursion. Python's Timsort on nearly sorted data is fast, but theoretically, sorting the strip at every level takes $O(N \log N)$ inside an $O(\log N)$ tree, yielding $O(N log^2 N)$ worst-case time.*
To achieve strict $O(N \log N)$, you must pre-sort all points by Y-coordinate once at the beginning, and carefully filter them into the left and right recursive calls.
Space complexity is bounded by $O(N)$ for array slicing/allocating the strip.

## Variants & optimizations

- **Sweep Line Algorithm:** You can solve this without divide and conquer! Maintain an active window of points sorted by Y-coordinate in a balanced BST. As a vertical line sweeps right, remove points that are further than `d` behind the line, and check the new point against the BST. Also strict $O(N \log N)$.

## Real-world applications

- **LIDAR Processing:** Rapidly finding the closest physical obstacles by analyzing massive point clouds generated by autonomous vehicle sensors.
