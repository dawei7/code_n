## Problem Statement

Chef is fascinated by number sequences and discovered a special sequence called the **Count-and-Say Sequence**.

The sequence is defined as follows:

- The **first term** is `"1"`.
- Each subsequent term is generated by applying **run-length encoding (RLE)** to the previous term:
  - For each group of consecutive identical digits, write the **count** followed by the **digit**.

For example:
- `"1"` → `"11"` (one `1`)
- `"11"` → `"21"` (two `1`s)
- `"21"` → `"1211"` (one `2`, one `1`)

You are given an integer $n$. Your task is to generate the **n-th term** of the sequence.

### Note:
Solve the problem **iteratively** to avoid stack overflow for larger values of $n$.

## Function Declaration

### Function Name
$countAndSay$ – This function generates the **n-th term** of the **Count-and-Say Sequence** using an **iterative approach**.

### Parameters

* $n$ : A positive integer representing the position in the Count-and-Say sequence.

### Return Value

* Returns a string representing the **n-th term** of the Count-and-Say Sequence.

## Constraints

- $1 \leq T \leq 50$
- $1 \leq n \leq 30$

# Problem Understanding

The **Count-and-Say Sequence** is a sequence of strings generated using a **run-length encoding rule**:

- Start with `"1"`.
- Each next term is generated by **reading aloud** (counting and saying) the previous term.

| n | Term       | Explanation                       |
| - | ---------- | --------------------------------- |
| 1 | `"1"`      | Base case                         |
| 2 | `"11"`     | One 1 → `"11"`                    |
| 3 | `"21"`     | Two 1s → `"21"`                   |
| 4 | `"1211"`   | One 2, One 1 → `"1211"`           |
| 5 | `"111221"` | One 1, One 2, Two 1s → `"111221"` |

---

# Approach

We can solve this **iteratively** (recommended) or **recursively** (simple but slow).

**Iterative Approach (Efficient)**

Instead of recursion, we build each term from the previous one **step by step**.

**Steps**:

Initialize result = `"1"`.

Loop from `i = 2` to `n`:

- Generate the next term from the previous one using run-length encoding.
- Update `result` each time.

**Function**: `nextTerm(s)``

Initialize:

- `count = 1`
- `res = ""`

For each character `s[i]`:

- If next character is same → increment count.
- Else:

   - Append count and character to `res`.

   - Reset count = 1.

---

# Time and Space Complexity

| Parameter | Complexity | Explanation                     |
| --------- | ---------- | ------------------------------- |
| **Time**  | O(M × n)   | M = average length of each term |
| **Space** | O(M)       | Storing each generated term     |

⚠️ Note: The length of the string grows roughly by 30–40% each step, but it’s still efficient for `n ≤ 30`.

---

# Key Insights

Recursive approach is elegant but **inefficient** for large n (due to deep recursion and repeated computation).

Iterative approach avoids stack overflow and is **memory-safe**.

Each term is **run-length encoded** version of the previous one.

Works efficiently up to `n = 30` as required
