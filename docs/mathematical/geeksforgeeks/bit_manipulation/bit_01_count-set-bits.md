# Formal Mathematical Specification: Count Set Bits (Brian Kernighan's Algorithm)

## 1. Definitions and Notation

Let $\mathbb{N}_0 = \mathbb{N} \cup \{0\}$ denote the set of non-negative integers. For a fixed word size $W \in \mathbb{N}$ (typically $W = 32$ or $W = 64$), we define the domain of our input as the ring of integers modulo $2^W$, denoted by:

$$\mathbb{Z}_{2^W} = \{0, 1, \dots, 2^W - 1\}$$

### 1.1 Binary Representation
For any integer $n \in \mathbb{Z}_{2^W}$, there exists a unique sequence of bits $\mathbf{b} = (b_{W-1}, b_{W-2}, \dots, b_0) \in \{0, 1\}^W$ such that:

$$n = \sum_{i=0}^{W-1} b_i 2^i$$

We define the projection function $[\cdot]_i : \mathbb{Z}_{2^W} \to \{0, 1\}$ to extract the $i$-th bit of $n$:

$$[n]_i = b_i$$

### 1.2 Hamming Weight
The Hamming weight (or population count) function $w : \mathbb{Z}_{2^W} \to \{0, 1, \dots, W\}$ maps an integer to the number of set bits ('1's) in its binary representation:

$$w(n) = \sum_{i=0}^{W-1} [n]_i$$

### 1.3 State Space
We model the execution of the algorithm as a discrete dynamical system over a state space $\mathcal{S}$ defined by:

$$\mathcal{S} = \mathbb{Z}_{2^W} \times \{0, 1, \dots, W\}$$

A state $s \in \mathcal{S}$ is represented as a tuple $(x, c)$, where:
* $x \in \mathbb{Z}_{2^W}$ is the current mutated value of the input.
* $c \in \{0, 1, \dots, W\}$ is the accumulator counting the set bits.

## 2. Algebraic Characterization

### 2.1 Bitwise Operations
For any $x, y \in \mathbb{Z}_{2^W}$, the bitwise AND operation, denoted by $x \land y$, is defined bitwise as:

$$[x \land y]_i = [x]_i \cdot [y]_i \quad \forall i \in \{0, 1, \dots, W-1\}$$

### 2.2 Lemma: The Rightmost Set Bit Erasure Identity
For any non-zero integer $x \in \mathbb{Z}_{2^W} \setminus \{0\}$, let $j \in \{0, 1, \dots, W-1\}$ be the index of the least significant set bit (LSB) of $x$. That is:

$$j = \min \{ i \in \{0, 1, \dots, W-1\} \mid [x]_i = 1 \}$$

Then, the algebraic transition $x \leftarrow x \land (x - 1)$ clears exactly the $j$-th bit of $x$ and leaves all other bits unchanged.

#### Proof:
By the definition of $j$, we can write the binary representation of $x$ as:

$$x = \sum_{i=j+1}^{W-1} [x]_i 2^i + 2^j + \sum_{i=0}^{j-1} 0 \cdot 2^i = \sum_{i=j+1}^{W-1} [x]_i 2^i + 2^j$$

Subtracting $1$ from $x$ yields:

$$x - 1 = \sum_{i=j+1}^{W-1} [x]_i 2^i + 2^j - 1$$

Using the geometric series identity $2^j - 1 = \sum_{i=0}^{j-1} 2^i$, we rewrite $x - 1$ as:

$$x - 1 = \sum_{i=j+1}^{W-1} [x]_i 2^i + \sum_{i=0}^{j-1} 2^i$$

Now, we evaluate the bitwise AND operation $y = x \land (x - 1)$ by analyzing the bits $[y]_i$ for three distinct intervals of $i$:

1. **For $i > j$:**
   $$[y]_i = [x]_i \land [x - 1]_i = [x]_i \land [x]_i = [x]_i$$

2. **For $i = j$:**
   $$[y]_j = [x]_j \land [x - 1]_j = 1 \land 0 = 0$$

3. **For $i < j$:**
   $$[y]_i = [x]_i \land [x - 1]_i = 0 \land 1 = 0$$

Combining these cases, the binary representation of $y$ is:

$$y = \sum_{i=j+1}^{W-1} [x]_i 2^i$$

Comparing this with the representation of $x$, we observe that:

$$w(y) = w(x) - 1$$

This completes the proof. $\blacksquare$

### 2.3 State Transition System
We define the state transition function $T : \mathcal{S} \to \mathcal{S}$ for any state $(x, c) \in \mathcal{S}$ where $x \neq 0$:

$$T(x, c) = (x \land (x - 1), c + 1)$$

The algorithm generates a sequence of states $(s_0, s_1, s_2, \dots)$ starting from the initial state:

$$s_0 = (n, 0)$$

The sequence is defined recursively by:

$$s_{k} = T(s_{k-1}) \quad \text{for } x_{k-1} \neq 0$$

### 2.4 Loop Invariant
For any state $s_k = (x_k, c_k)$ generated by the algorithm, the following invariant holds:

$$\mathcal{I}(s_k) \iff w(n) = w(x_k) + c_k$$

#### Proof by Induction:
* **Base Case ($k = 0$):**
  $$w(x_0) + c_0 = w(n) + 0 = w(n)$$
  The invariant holds.

* **Inductive Step:**
  Assume $\mathcal{I}(s_k)$ holds for some $k \ge 0$, and $x_k \neq 0$. The next state is $s_{k+1} = (x_{k+1}, c_{k+1}) = (x_k \land (x_k - 1), c_k + 1)$.
  Using the Lemma proved in Section 2.2, we have $w(x_{k+1}) = w(x_k) - 1$.
  Therefore:
  $$w(x_{k+1}) + c_{k+1} = (w(x_k) - 1) + (c_k + 1) = w(x_k) + c_k$$
  By the inductive hypothesis, $w(x_k) + c_k = w(n)$, hence:
  $$w(x_{k+1}) + c_{k+1} = w(n)$$
  The invariant $\mathcal{I}(s_{k+1})$ holds. $\blacksquare$

### 2.5 Termination and Correctness
Since $w(x_k) \in \mathbb{N}_0$ and $w(x_{k+1}) = w(x_k) - 1$, the sequence of Hamming weights $\{w(x_k)\}_{k=0}^{\infty}$ is strictly decreasing:

$$w(x_k) = w(n) - k$$

Because $w(x_k) \ge 0$ for all $k$, the sequence must terminate at some finite step $K$ where $x_K = 0$.
The termination condition $x_K = 0$ implies $w(x_K) = 0$.
Substituting this into the loop invariant $\mathcal{I}(s_K)$:

$$w(n) = w(x_K) + c_K = 0 + c_K = c_K$$

Thus, upon termination, the accumulator $c_K$ contains exactly the Hamming weight of the input $n$.

## 3. Complexity Analysis

### 3.1 Time Complexity
Let $K = w(n)$ be the number of set bits in the input $n \in \mathbb{Z}_{2^W}$.

* **Number of Iterations:** The loop terminates when $x_k = 0$. Since $w(x_k) = w(n) - k$, the termination condition $w(x_K) = 0$ is met if and only if $K = w(n)$ iterations are executed.
* **Work per Iteration:** Each iteration performs a constant number of primitive operations:
  1. One subtraction: $x_k - 1$
  2. One bitwise AND: $x_k \land (x_k - 1)$
  3. One addition: $c_k + 1$
  4. One assignment and comparison: $x_{k+1} \leftarrow x_k \land (x_k - 1)$ and checking if $x_{k+1} \neq 0$.

Under the Word RAM model, each of these operations takes $O(1)$ time. Thus, the total time complexity $T(n)$ is:

$$T(n) = \sum_{k=1}^{K} O(1) = \Theta(K)$$

* **Best-Case Complexity:** Occurs when $n = 0$, yielding $K = 0$.
  $$T_{\text{best}}(n) = \Theta(1)$$
* **Worst-Case Complexity:** Occurs when all bits are set ($n = 2^W - 1$), yielding $K = W$.
  $$T_{\text{worst}}(n) = \Theta(W)$$

For a fixed word size $W$ (e.g., $W = 32$ or $W = 64$), the time complexity is bounded by $O(1)$ relative to the magnitude of $n$, but asymptotically relative to the Hamming weight, it is strictly $\Theta(K)$.

### 3.2 Space Complexity
The algorithm is executed in-place and requires storage only for the state variables $(x, c) \in \mathcal{S}$.

* **Auxiliary Space:** The variables $x$ and $c$ require:
  $$\text{Bits}(x) = W$$
  $$\text{Bits}(c) = \lceil \log_2(W + 1) \rceil$$
  Since $W$ is a fixed hardware parameter, the auxiliary space complexity is:
  $$S_{\text{aux}}(n) = O(1) \text{ words}$$

* **Total Space:** Including the input $n$, the total space complexity is:
  $$S_{\text{total}}(n) = O(1) \text{ words}$$
