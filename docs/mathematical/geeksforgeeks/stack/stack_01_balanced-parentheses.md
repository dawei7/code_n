# Formal Mathematical Specification: Balanced Parentheses

## 1. Definitions and Notation

Let $\Sigma = \{'(', ')', '[', ']', '{', '}'\}$ be the alphabet of bracket characters. We define the set of opening brackets as $\Sigma_{open} = \{'(', '[', '{'\}$ and the set of closing brackets as $\Sigma_{close} = \{')', ']', '\}'\}$. 

We define a bijection $\mu: \Sigma_{close} \to \Sigma_{open}$ representing the pairing relationship:
$$\mu(')') = '(', \quad \mu(']') = '[', \quad \mu('\}') = '\{'$$

Let $s = s_1 s_2 \dots s_n$ be a string of length $n$ where $s_i \in \Sigma$. 
The state of the algorithm is represented by a stack $\sigma \in \Sigma_{open}^*$, where $\Sigma_{open}^*$ is the set of all finite sequences (strings) over $\Sigma_{open}$. We denote the empty stack as $\epsilon$. The operation $\text{push}(\sigma, c)$ denotes the concatenation $\sigma \cdot c$, and $\text{pop}(\sigma)$ denotes the prefix $\sigma'$ such that $\sigma = \sigma' \cdot x$ for some $x \in \Sigma_{open}$.

The algorithm defines a function $f: \Sigma^* \to \{0, 1\}$, where $1$ denotes a valid (balanced) string and $0$ denotes an invalid string.

## 2. Algebraic Characterization

The validity of the string $s$ is governed by the state transition function $\delta: (\Sigma_{open}^*, \Sigma) \to \Sigma_{open}^* \cup \{\bot\}$, where $\bot$ represents an invalid state. For a character $c \in \Sigma$ and stack $\sigma$:

$$\delta(\sigma, c) = 
\begin{cases} 
\sigma \cdot c & \text{if } c \in \Sigma_{open} \\
\sigma' & \text{if } c \in \Sigma_{close}, \sigma = \sigma' \cdot \mu(c) \\
\bot & \text{otherwise}
\end{cases}$$

**Loop Invariant:**
Let $\sigma_i$ be the state of the stack after processing the prefix $s_{1 \dots i}$. The algorithm is correct if and only if:
1. For all $i \in \{1, \dots, n\}$, $\sigma_i \neq \bot$.
2. $\sigma_n = \epsilon$.

This can be viewed as a reduction process on a Dyck language $D_k$. The string $s$ is balanced if it belongs to the language generated by the context-free grammar $G = (V, \Sigma, R, S)$ with $V=\{S\}$, $R=\{S \to SS, S \to (S), S \to [S], S \to \{S\}, S \to \epsilon\}$. The stack-based algorithm effectively performs a bottom-up parse of the string to verify membership in $L(G)$.

## 3. Complexity Analysis

### Time Complexity
The algorithm processes each character $s_i$ in the input string exactly once. For each $s_i$, the algorithm performs:
1. A membership test in $\Sigma_{open}$ or $\Sigma_{close}$, which is $O(1)$ given a fixed alphabet size $|\Sigma| = 6$.
2. A stack operation ($\text{push}$ or $\text{pop}$), which is $O(1)$ in a standard array-based or linked-list implementation.
3. A dictionary lookup for $\mu(c)$, which is $O(1)$.

The total time complexity $T(n)$ is given by the summation:
$$T(n) = \sum_{i=1}^{n} \Theta(1) = \Theta(n)$$
Thus, the algorithm is strictly $O(n)$.

### Space Complexity
The space complexity $S(n)$ is determined by the maximum depth of the stack $\sigma$. In the worst-case scenario, the input string consists entirely of opening brackets (e.g., $s = ('^n$), requiring the stack to store $n$ elements. 

The auxiliary space is:
$$S(n) = \max_{0 \le i \le n} |\sigma_i|$$
Since $|\sigma_i| \le i$, we have $S(n) = O(n)$. 

In the best-case scenario, where the string is invalid immediately (e.g., $s_1 \in \Sigma_{close}$), the space complexity is $O(1)$. However, the asymptotic upper bound remains $O(n)$ to account for the worst-case input.
