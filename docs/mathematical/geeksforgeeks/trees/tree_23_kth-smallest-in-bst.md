# Formal Mathematical Specification: K-th Smallest in BST

## 1. Definitions and Notation
Let $T$ be a BST with $n$ nodes. We seek the node $v$ whose value has rank $k$ in the sorted sequence of $V$.

## 2. Algebraic Characterization
By the BST property, the sequence generated by the in-order traversal $\mathcal{I}(T)$ is monotonic.
The required node is precisely the $k$-th element of $\mathcal{I}(T)$.

If nodes are augmented with a size attribute $S(v) = 1 + S(v_L) + S(v_R)$, the search can be directed recursively without full traversal:
$$ \text{Find}(T, k) = \begin{cases} 
T & \text{if } k = S(T_L) + 1 \\
\text{Find}(T_L, k) & \text{if } k \leq S(T_L) \\
\text{Find}(T_R, k - S(T_L) - 1) & \text{if } k > S(T_L) + 1
\end{cases} $$

## 3. Complexity Analysis
- **Time Complexity:** Explicit in-order traversal takes $O(\mathcal{H}(T) + k)$ time. Augmented size search operates in strict $O(\mathcal{H}(T))$ time.
- **Space Complexity:** $O(\mathcal{H}(T))$.
