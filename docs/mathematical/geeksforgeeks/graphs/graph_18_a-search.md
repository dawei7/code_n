# Formal Mathematical Specification: A* Search Algorithm

## 1. Definitions and Notation

Let $G = (V, E)$ be a weighted, directed graph where $V$ is a finite set of vertices and $E \subseteq V \times V$ is a set of edges. Let $w: E \to \mathbb{R}^+$ be a cost function assigning a positive weight to each edge.

*   **Start and Target:** We define a source vertex $s \in V$ and a target vertex $t \in V$.
*   **Path:** A path $P$ from $s$ to $t$ is a sequence of vertices $(v_0, v_1, \dots, v_k)$ such that $v_0 = s$, $v_k = t$, and $(v_i, v_{i+1}) \in E$ for all $0 \le i < k$.
*   **Cost Function:** The cost of a path $P$ is defined as $c(P) = \sum_{i=0}^{k-1} w(v_i, v_{i+1})$.
*   **Optimal Path:** The shortest path distance $d^*(s, t)$ is defined as $\min \{c(P) \mid P \text{ is a path from } s \text{ to } t\}$.
*   **Heuristic Function:** Let $h: V \to \mathbb{R}_{\ge 0}$ be a heuristic function such that $h(t) = 0$.
*   **State Variables:**
    *   $g(n)$: The cost of the path from $s$ to $n$ found by the algorithm.
    *   $f(n) = g(n) + h(n)$: The estimated total cost of a path from $s$ to $t$ passing through $n$.
    *   $\mathcal{O}$: The "Open Set" (priority queue), containing nodes discovered but not yet expanded, ordered by $f(n)$.
    *   $\mathcal{C}$: The "Closed Set," containing nodes already expanded.

## 2. Algebraic Characterization

The correctness of A* relies on the properties of the heuristic $h(n)$.

### Admissibility
A heuristic $h$ is **admissible** if for all $n \in V$:
$$h(n) \le d^*(n, t)$$
where $d^*(n, t)$ is the true shortest path distance from $n$ to $t$. If $h$ is admissible, A* is guaranteed to find an optimal path.

### Consistency (Monotonicity)
A heuristic $h$ is **consistent** if for every node $n$ and every successor $n'$ of $n$ generated by an edge $(n, n')$ with cost $w(n, n')$:
$$h(n) \le w(n, n') + h(n')$$
Consistency implies admissibility. Under consistency, the $f$-values of nodes expanded by A* are non-decreasing, ensuring that when a node $n$ is expanded, $g(n) = d^*(s, n)$.

### The Expansion Invariant
At any iteration, let $g(n)$ be the current best-known distance from $s$ to $n$. The algorithm maintains the following invariant:
For any node $n \in \mathcal{O}$, $g(n)$ is the shortest path distance from $s$ to $n$ found so far. The algorithm selects $n \in \mathcal{O}$ such that:
$$n = \arg\min_{u \in \mathcal{O}} \{g(u) + h(u)\}$$
Upon expansion of $n$, for each neighbor $v$ of $n$, the algorithm performs a relaxation step:
$$g(v) = \min(g(v), g(n) + w(n, v))$$
If $g(v)$ is updated, $v$ is inserted into $\mathcal{O}$ with priority $f(v) = g(v) + h(v)$.

## 3. Complexity Analysis

### Time Complexity
The time complexity is governed by the number of priority queue operations. Each edge $(u, v) \in E$ is relaxed at most once if $h$ is consistent.
*   **Total Work:** The algorithm performs at most $|V|$ `extract-min` operations and at most $|E|$ `decrease-key` (or `insert`) operations.
*   **Asymptotic Bound:** Using a Fibonacci heap, the complexity is $O(E + V \log V)$. Using a standard binary heap, it is $O(E \log V)$.
*   **Heuristic Influence:** The effective branching factor $b^*$ determines the number of nodes expanded. If $h(n) = 0$ (Dijkstra), the complexity is $O(E \log V)$. If $h(n) = d^*(n, t)$ (perfect heuristic), the complexity is $O(V)$ as the algorithm explores only the nodes on the optimal path.

### Space Complexity
The space complexity is dominated by the storage of the graph and the auxiliary data structures:
*   **Graph Storage:** $O(V + E)$ for the adjacency list.
*   **Auxiliary Space:** The priority queue $\mathcal{O}$ and the distance map $g$ store at most $O(V)$ nodes.
*   **Total Space:** $O(V + E)$, which simplifies to $O(V)$ in sparse graphs where $E \approx V$.
