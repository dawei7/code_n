# Formal Mathematical Specification: Rat in a Maze

## 1. Definitions and Notation

Let $N \in \mathbb{N}^+$ denote the dimension of a square grid. We define the coordinate index set as $I_N = \{0, 1, \dots, N-1\}$. The spatial domain of the maze is represented by the Cartesian product:

$$\mathcal{C} = I_N \times I_N$$

The maze is formally defined as a binary matrix $M \in \{0, 1\}^{N \times N}$, where for any coordinate $u = (r, c) \in \mathcal{C}$:
* $M(u) = 1$ represents an open, traversable cell.
* $M(u) = 0$ represents a blocked cell (a wall).

We define the set of traversable vertices $V \subseteq \mathcal{C}$ as:

$$V = \{ u \in \mathcal{C} \mid M(u) = 1 \}$$

Let $\|\cdot\|_1$ denote the $L_1$ norm (Manhattan distance) on $\mathbb{Z}^2$. For any two coordinates $u = (r_1, c_1)$ and $v = (r_2, c_2)$, the distance is given by:

$$\|u - v\|_1 = |r_1 - r_2| + |c_1 - c_2|$$

We define the set of directed edges $E \subseteq V \times V$ representing valid 4-directional moves within the open spaces of the maze:

$$E = \{ (u, v) \in V \times V \mid \|u - v\|_1 = 1 \}$$

The maze is thus modeled as an unweighted, undirected grid graph $G = (V, E)$. 

### Paths and Directions
Let $s = (0, 0) \in \mathcal{C}$ be the source vertex, and $t = (N-1, N-1) \in \mathcal{C}$ be the target vertex. We assume $s, t \in V$; if $M(s) = 0$ or $M(t) = 0$, the set of valid paths is trivially empty.

A **simple path** $P$ of length $k$ from $s$ to $t$ in $G$ is a sequence of vertices $P = (v_0, v_1, \dots, v_k)$ satisfying:
1. $v_0 = s$ and $v_k = t$.
2. $(v_i, v_{i+1}) \in E$ for all $0 \le i < k$.
3. $v_i \neq v_j$ for all $0 \le i < j \le k$ (the self-avoiding property).

Let $\mathcal{P}_{s, t}$ denote the set of all such simple paths from $s$ to $t$.

To represent paths as directional strings, we define the set of move directions $\mathcal{D} = \{\text{'U'}, \text{'D'}, \text{'L'}, \text{'R'}\}$ and a bijective mapping $\phi: E \to \mathcal{D}$ defined by:

$$\phi((r, c), (r', c')) = \begin{cases} 
\text{'U'} & \text{if } r' = r - 1 \\
\text{'D'} & \text{if } r' = r + 1 \\
\text{'L'} & \text{if } c' = c - 1 \\
\text{'R'} & \text{if } c' = c + 1 
\end{cases}$$

For any simple path $P = (v_0, v_1, \dots, v_k) \in \mathcal{P}_{s, t}$, its corresponding string representation $S(P) \in \mathcal{D}^k$ is the sequence:

$$S(P) = \left( \phi(v_0, v_1), \phi(v_1, v_2), \dots, \phi(v_{k-1}, v_k) \right)$$

The objective of the **Rat in a Maze** algorithm is to construct the set of all path strings:

$$\mathcal{S}_{s, t} = \{ S(P) \mid P \in \mathcal{P}_{s, t} \}$$

## 2. Algebraic Characterization

The backtracking algorithm systematically explores the state space of self-avoiding walks on $G$ using depth-first search (DFS) with pruning. 

### State Space Representation
A state $\sigma$ in the backtracking search is represented by a triple:

$$\sigma = (u, \mathcal{V}, \pi)$$

where:
* $u \in V$ is the current vertex.
* $\mathcal{V} \subseteq V$ is the set of vertices visited along the current path (to prevent cycles).
* $\pi \in V^*$ is the sequence of vertices representing the active path from $s$ to $u$.

Let $\Sigma = V \times \mathcal{P}(V) \times V^*$ be the state space, where $\mathcal{P}(V)$ denotes the power set of $V$.

### Recursive Formulation
We define the search function $f: \Sigma \to \mathcal{P}(V^*)$ which, given a state $(u, \mathcal{V}, \pi)$, returns the set of all valid simple paths from $s$ to $t$ that extend the prefix $\pi$. 

The function $f$ is defined recursively as:

$$f(u, \mathcal{V}, \pi) = \begin{cases}
\{\pi\} & \text{if } u = t \\
\emptyset & \text{if } u \neq t \text{ and } \text{Adj}(u) \setminus \mathcal{V} = \emptyset \\
\bigcup_{w \in \text{Adj}(u) \setminus \mathcal{V}} f(w, \mathcal{V} \cup \{w\}, \pi \cdot w) & \text{otherwise}
\end{cases}$$

where:
* $\text{Adj}(u) = \{ v \in V \mid (u, v) \in E \}$ is the set of open neighbors of $u$.
* $\pi \cdot w$ denotes the concatenation of vertex $w$ to the sequence $\pi$.

The initial call to the algorithm is:

$$\mathcal{P}_{s, t} = f(s, \{s\}, (s))$$

### Backtracking and State Restoration
The algebraic formulation of $f$ naturally captures the backtracking mechanism. The set union operator $\bigcup$ over the unvisited neighbors $\text{Adj}(u) \setminus \mathcal{V}$ implies that each branch is evaluated independently. 

When the recursion unwinds from a branch $w$, the visited set $\mathcal{V} \cup \{w\}$ is discarded, restoring the visited context to $\mathcal{V}$ for the next sibling branch in $\text{Adj}(u) \setminus \mathcal{V}$. This mathematical scoping corresponds directly to the imperative state restoration step:

$$\text{visited}[r][c] = \text{False}$$

## 3. Complexity Analysis

### Time Complexity

To establish the worst-case time complexity, we analyze the size of the state-space tree generated by the recursive function $f$.

#### Theorem 1
*The worst-case time complexity of the Rat in a Maze algorithm on an $N \times N$ grid is $O(4^{N^2})$.*

#### Proof:
Let $G = (V, E)$ be a complete grid graph where $M(u) = 1$ for all $u \in \mathcal{C}$. Thus, $|V| = N^2$. 

1. **Upper Bound via Simple Paths:**
   The backtracking algorithm generates all self-avoiding walks (SAWs) starting from $s$. Any self-avoiding walk on $G$ can have a maximum length of $|V| = N^2$ vertices.
   At the start vertex $s$, there are at most 4 possible directions to move. For any subsequent vertex $u$ in the path, the incoming edge occupies one of the 4 directions, leaving at most 3 potential directions for exploration:
   
   $$\text{Branching Factor } b \le 3$$

   Let $T(d)$ denote the number of paths of length $d$ starting from $s$. We can bound $T(d)$ by:
   
   $$T(d) \le 4 \cdot 3^{d-1} \quad \text{for } d \ge 1$$

   The total number of states visited in the search tree of maximum depth $D = N^2$ is bounded by the sum of the nodes at each depth:
   
   $$\sum_{d=0}^{N^2} T(d) \le 1 + \sum_{d=1}^{N^2} 4 \cdot 3^{d-1} = 1 + 4 \left( \frac{3^{N^2} - 1}{3 - 1} \right) = 2 \cdot 3^{N^2} - 1$$

2. **Asymptotic Complexity:**
   Since $2 \cdot 3^{N^2} - 1 \in O(3^{N^2})$, and $O(3^{N^2}) \subset O(4^{N^2})$, the worst-case time complexity is bounded by $O(4^{N^2})$. This loose upper bound of $O(4^{N^2})$ is conventionally cited in literature to represent the unconstrained 4-way branching factor at every cell.
   
   Thus, the worst-case time complexity is $\Theta(3^{N^2})$ or $O(4^{N^2})$. $\blacksquare$

### Space Complexity

#### Theorem 2
*The auxiliary space complexity of the algorithm is $O(N^2)$.*

#### Proof:
The auxiliary space consumed by the algorithm consists of two primary components:
1. **The Recursion Stack:** 
   The maximum depth of the recursion tree is bounded by the maximum length of a simple path in $G$. Since the graph has $|V| = N^2$ vertices, any simple path can contain at most $N^2$ vertices. Therefore, the maximum depth of the call stack is $N^2$, requiring $O(N^2)$ space.

2. **The State Storage:**
   * The `visited` lookup table is a boolean matrix of size $N \times N$, requiring $\Theta(N^2)$ space.
   * The current path sequence $\pi$ stores at most $N^2$ vertices, requiring $O(N^2)$ space.

Summing these components:

$$\text{Space}_{\text{aux}}(N) = \underbrace{O(N^2)}_{\text{Stack Depth}} + \underbrace{\Theta(N^2)}_{\text{Visited Matrix}} + \underbrace{O(N^2)}_{\text{Path Storage}} = O(N^2)$$

If we include the output space required to store all valid paths, the total space complexity is $O(N^2 \cdot 3^{N^2})$ in the worst case, as there can be up to $O(3^{N^2})$ paths, each of length $O(N^2)$. $\blacksquare$
