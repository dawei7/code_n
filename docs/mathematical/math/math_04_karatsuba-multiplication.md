# Formal Mathematical Specification: Karatsuba Multiplication

## 1. Definitions and Notation

Let $X, Y \in \mathbb{Z}^+$ be two positive integers represented in base $B$ (typically $B=10$). Let $n$ be the number of digits of the larger integer, defined as $n = \max(\lfloor \log_B X \rfloor + 1, \lfloor \log_B Y \rfloor + 1)$.

We define the decomposition of $X$ and $Y$ relative to a split point $m = \lceil n/2 \rceil$:
- $X = A \cdot B^m + B_{low}$
- $Y = C \cdot B^m + D_{low}$

Where:
- $A = \lfloor X / B^m \rfloor$ and $B_{low} = X \pmod{B^m}$
- $C = \lfloor Y / B^m \rfloor$ and $D_{low} = Y \pmod{B^m}$

The domain of the algorithm is the set of natural numbers $\mathbb{N}$, and the codomain is the product $P = X \cdot Y$. The state space $\mathcal{S}$ is defined by the recursion tree generated by the recursive decomposition of the operands until the base case $n < 2$ is reached.

## 2. Algebraic Characterization

The product $P = X \cdot Y$ is expanded via the distributive property:
$$P = (A \cdot B^m + B_{low})(C \cdot B^m + D_{low})$$
$$P = (A \cdot C) \cdot B^{2m} + (A \cdot D_{low} + B_{low} \cdot C) \cdot B^m + (B_{low} \cdot D_{low})$$

Let the three fundamental products be defined as:
1. $Z_2 = A \cdot C$
2. $Z_0 = B_{low} \cdot D_{low}$
3. $Z_1 = (A + B_{low}) \cdot (C + D_{low})$

By expanding $Z_1$, we observe:
$$Z_1 = AC + AD_{low} + B_{low}C + B_{low}D_{low}$$
$$Z_1 = Z_2 + (AD_{low} + B_{low}C) + Z_0$$

Thus, the middle term $(AD_{low} + B_{low}C)$ can be isolated without explicit computation of the individual cross-products:
$$(AD_{low} + B_{low}C) = Z_1 - Z_2 - Z_0$$

Substituting this into the expansion of $P$, we obtain the Karatsuba identity:
$$P = Z_2 \cdot B^{2m} + (Z_1 - Z_2 - Z_0) \cdot B^m + Z_0$$

This identity reduces the number of recursive multiplications from four to three, which is the fundamental algebraic transition governing the algorithm's efficiency.

## 3. Complexity Analysis

### Time Complexity
The algorithm follows a divide-and-conquer paradigm. The work performed at each level of the recursion consists of additions, subtractions, and shifts (multiplications by powers of $B$), which are $O(n)$ operations. The recurrence relation for the time complexity $T(n)$ is:
$$T(n) = 3T(n/2) + O(n)$$

Applying the Master Theorem for divide-and-conquer recurrences of the form $T(n) = aT(n/b) + f(n)$:
- $a = 3$
- $b = 2$
- $f(n) = O(n^1)$

Since $\log_b a = \log_2 3 \approx 1.585$, and $f(n) = O(n^c)$ where $c < \log_b a$, the complexity is dominated by the recursive calls:
$$T(n) = \Theta(n^{\log_2 3}) \approx O(n^{1.585})$$

### Space Complexity
The space complexity $S(n)$ is determined by the maximum depth of the recursion stack and the auxiliary storage required for the intermediate products $Z_0, Z_1, Z_2$.

1. **Recursion Depth:** The problem size is halved at each step, leading to a recursion tree depth of $d = \lceil \log_2 n \rceil$.
2. **Stack Frame:** Each frame stores a constant number of variables ($A, B_{low}, C, D_{low}, Z_0, Z_1, Z_2$).
3. **Auxiliary Space:** While the total number of bits stored across all levels of the stack is $O(n)$, the *active* memory at any point in the depth-first traversal is $O(n)$ due to the storage of intermediate sums. However, if we consider the stack depth specifically as the primary auxiliary space constraint, it is $O(\log n)$. 

Thus, the auxiliary space complexity is $O(n)$ for the storage of numbers at each level, but the recursion stack depth is $O(\log n)$. In standard implementations, the total space complexity is $O(n)$.
