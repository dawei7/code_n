## Problem Description

### Goal

Strings `s1` and `s2` have the same length. For every index `i`, characters `s1[i]` and `s2[i]` are declared equivalent. These declarations form an equivalence relation: every character is equivalent to itself, equivalence is symmetric, and equivalence is transitive.

Using all equivalence information supplied by the paired strings, replace each character of `baseStr` by any equivalent lowercase English letter. Return the lexicographically smallest string that can be obtained. Choices at different positions are independent, but each may use equivalences implied through an entire transitive chain rather than only a directly paired character.

### Function Contract

**Inputs**

- `s1` and `s2`: lowercase English strings of equal length $P$, where $1 \le P \le 1000$.
- `baseStr`: a lowercase English string of length $B$, where $1 \le B \le 1000$.
- Let $A=26$ be the lowercase English alphabet size.

**Return value**

- The lexicographically smallest string equivalent to `baseStr` under the relation generated by corresponding characters of `s1` and `s2`.

### Examples

**Example 1**

- Input: `s1 = "parker", s2 = "morris", baseStr = "parser"`
- Output: `"makkek"`
- Explanation: The equivalence classes include `[m, p]`, `[a, o]`, `[k, r, s]`, and `[e, i]`; each base character uses its class minimum.

**Example 2**

- Input: `s1 = "hello", s2 = "world", baseStr = "hold"`
- Output: `"hdld"`

**Example 3**

- Input: `s1 = "leetcode", s2 = "programs", baseStr = "sourcecode"`
- Output: `"aauaaaaada"`
