## Problem Description
[Open the original LeetCode problem](https://leetcode.com/problems/kth-smallest-amount-with-single-denomination-combination/).

### Goal
Given a collection of coin denominations, determine the k-th smallest positive integer that can be represented as a multiple of at least one of these denominations. Since the set of such multiples is infinite, we seek the k-th value in the sorted sequence of all unique multiples generated by the provided coins.

### Function Contract
**Inputs**

- `coins`: A list of integers representing the available coin denominations.
- `k`: An integer representing the rank of the value to find.

**Return value**

- An integer representing the k-th smallest value that is a multiple of at least one coin in the input list.

### Examples
**Example 1**

- Input: `coins = [3, 6, 9], k = 3`
- Output: `9`
- Explanation: The multiples are 3, 6, 9, 12, 15, 18... The 3rd smallest is 9.

**Example 2**

- Input: `coins = [5, 2], k = 7`
- Output: `12`
- Explanation: The sorted multiples are 2, 4, 5, 6, 8, 10, 12... The 7th smallest is 12.

**Example 3**

- Input: `coins = [1], k = 1`
- Output: `1`

---

## Solution
### Approach
The problem is solved using **Binary Search on the Answer** combined with the **Inclusion-Exclusion Principle**.
1. The range for binary search is `[min(coins), k * min(coins)]`.
2. For a candidate value `x`, we count how many multiples exist $\le x$ using the Inclusion-Exclusion Principle to avoid double-counting multiples of the Least Common Multiple (LCM) of subsets of coins.
3. The count function uses recursion or bitmasking to calculate the union of multiples.

### Complexity Analysis
- **Time Complexity**: $O(2^N \cdot \log(\text{max\_val}) \cdot \log(\text{max\_coin}))$, where $N$ is the number of coins, $\text{max\_val}$ is the result, and $\text{max\_coin}$ is the largest denomination.
- **Space Complexity**: $O(N)$ to store the coins and recursion stack.

```python
import math
from typing import List

def solve(coins: List[int], k: int) -> int:
    """
    Finds the k-th smallest multiple using binary search and inclusion-exclusion.
    """
    coins = sorted(set(coins))
    reduced: list[int] = []
    for coin in coins:
        if not any(coin % existing == 0 for existing in reduced):
            reduced.append(coin)
    coins = reduced

    def get_lcm(a, b):
        return abs(a * b) // math.gcd(a, b)

    high = min(coins) * k
    terms: list[tuple[int, int]] = []

    def build_terms(index: int, current_lcm: int, parity: int) -> None:
        for pos in range(index, len(coins)):
            next_lcm = get_lcm(current_lcm, coins[pos])
            if next_lcm > high:
                continue
            terms.append((next_lcm, parity))
            build_terms(pos + 1, next_lcm, -parity)

    build_terms(0, 1, 1)

    def count_multiples(mid: int) -> int:
        count = 0
        for lcm_val, sign in terms:
            count += sign * (mid // lcm_val)
        return count

    low = min(coins)
    ans = high

    while low <= high:
        mid = (low + high) // 2
        if count_multiples(mid) >= k:
            ans = mid
            high = mid - 1
        else:
            low = mid + 1

    return ans
```
