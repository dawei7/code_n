## Problem Description

### Goal

A Fibonacci tree is defined by its order. `order(0)` is empty, and `order(1)` contains one node. For $n\ge2$, `order(n)` has a new root whose left subtree is `order(n - 2)` and whose right subtree is `order(n - 1)`.

Alice and Bob play on `order(n)`, with Alice moving first. A move selects one node and removes that node together with all of its descendants. Eventually a player has no safe choice and is forced to delete the original root; that player loses. Assuming optimal play from both sides, determine whether Alice wins.

### Function Contract

**Inputs**

- `n`: the Fibonacci-tree order, where $1 \le n \le 100$.

**Return value**

Return `true` when the initial position is winning for Alice under optimal play, and `false` when Bob can force Alice to remove the root.

### Examples

**Example 1**

- Input: `n = 3`
- Output: `true`
- Explanation: Alice can remove one leaf so that after the remaining safe removals, Bob is the player forced to take the root.

**Example 2**

- Input: `n = 1`
- Output: `false`
- Explanation: The tree contains only the root, so Alice loses immediately.

**Example 3**

- Input: `n = 2`
- Output: `true`
- Explanation: Alice removes the only non-root node, leaving Bob with the losing root move.

### Required Complexity

- **Time:** $O(1)$
- **Space:** $O(1)$

<summary>Approach</summary>

#### General

**Interpret the protected root as the terminal position.** Removing any proper subtree is a safe move. Once no proper subtree remains, the player to move is forced to remove the root and loses, so that state is a losing impartial-game position.

**Exploit the recursive game-state cycle.** The two branches of `order(n)` are exactly the game states generated by orders $n-2$ and $n-1$. Applying the standard impartial-game transition to these recursively paired branches produces a complete state signature that repeats every six orders. Checking the first six nonempty orders gives:

| $n\bmod 6$ | $0$ | $1$ | $2$ | $3$ | $4$ | $5$ |
|---:|---:|---:|---:|---:|---:|---:|
| Winner | Alice | Bob | Alice | Alice | Alice | Alice |

Because the transition for the next order depends on the preceding Fibonacci-order signatures, the repeated pair of signatures after six steps makes the sequence periodic by induction.

**Test the only losing residue.** Bob wins exactly when $n\equiv1\pmod6$. Therefore, Alice wins precisely when `n % 6 != 1`, requiring neither construction of the tree nor simulation of moves.

#### Complexity detail

One remainder operation and one comparison take $O(1)$ time and $O(1)$ space, independent of the number of nodes represented by the Fibonacci-tree order.

#### Alternatives and edge cases

- **Construct the Fibonacci tree:** Its number of nodes grows exponentially with $n$, making explicit construction impossible near order $100$.
- **Dynamic programming over orders:** Computing the game signatures through order $n$ takes $O(n)$ time and can reveal the period, but the proven six-order cycle reduces evaluation to one residue.
- Order $1$ is the smallest and only immediately losing example because it contains no proper subtree.
- Orders $7,13,19,\ldots$ repeat the losing residue even though their represented trees are much larger.
- Order $100$ is evaluated directly by its remainder; tree size never enters the computation.
